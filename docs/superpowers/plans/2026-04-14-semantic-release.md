# Semantic Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Automate npm publishing and GitHub Release creation for `@quydoan/myskills` using `semantic-release` on every push to `main`.

**Architecture:** Install `semantic-release` as a dev dependency, configure it via `release.config.js`, update `package.json` for public scoped publishing, and add a GitHub Actions workflow that runs `npx semantic-release` on pushes to `main`.

**Tech Stack:** semantic-release, GitHub Actions, npm registry

### Task 4: Add GitHub Actions release workflow

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: Create the release workflow**

```yaml
name: Release

on:
  push:
    branches:
      - main

permissions:
  contents: write
  issues: write
  pull-requests: write

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v5

      - uses: actions/setup-node@v5
        with:
          node-version: "22"
          cache: "npm"

      - run: npm ci

      - run: npx semantic-release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
```

Notes on the `permissions` block:
- `contents: write` — needed to create GitHub Releases and push version tags
- `issues: write` — needed for semantic-release to comment on issues referenced in commits
- `pull-requests: write` — needed for semantic-release to comment on associated PRs

- [ ] **Step 2: Validate the workflow YAML**

Run: `node -e "import('yaml').then(m => { const fs = require('fs'); const doc = m.parse(fs.readFileSync('.github/workflows/release.yml', 'utf8')); console.log('Trigger:', JSON.stringify(doc.on)); console.log('Jobs:', Object.keys(doc.jobs)); })"`

If that command is awkward with ESM, use:
Run: `npx yaml .github/workflows/release.yml` or simply `cat .github/workflows/release.yml` to visually verify.

Expected: Valid YAML with trigger on `push` to `main`, one job `release` with 4 steps.

- [ ] **Step 3: Commit**

```bash
git add .github/workflows/release.yml
git commit -m "ci: add semantic-release GitHub Actions workflow"
```

### Task 5: Dry-run verification

This task verifies the full setup locally before pushing.

- [ ] **Step 1: Run semantic-release in dry-run mode**

Run: `GITHUB_TOKEN=fake NPM_TOKEN=fake npx semantic-release --dry-run`

Expected: semantic-release runs through its analysis phase. It will either:
- Report "no release" if the current commits don't trigger a release (this is fine — it means config loaded correctly)
- Report what version it would publish (also fine)
- It should NOT error with config parsing or plugin loading failures

If you see errors about missing plugins or config, fix `release.config.js` before proceeding.

- [ ] **Step 2: Commit (if any fixes were needed)**

Only if Step 1 required changes:
